cara upload ke github

  • pertama masuk ke akun github
  • buat repo-lokal di komputer
  • kemudian buka bash git, buka folder repo yang telah di buat
  • kemudian ketik kan "git init", proses ini akan memantau aktifitas yang ada di repositori lokal
  • kemudian buka github di browser
  • masuk akun
  • kemudian buat repositori baru
  • buat nama repositori, tetapi jika ingin menjadikan repositori itu sebagai web hosting
  • buat repo baru dengan nama username.github.io, contoh ilhamharahap1.github.io
  • jika ingin membuat domain ke dua tidak perlu membuat nama sesua ketentuan, cukup buat nama repo saja
  • kemudian jangan centang readme agar tampilan memberi link untuk remote
  • kemudian buka bash kembali
  • ketik, git remote add nama remote kemudian link yang di berikan
  • contoh : git remote add origin(nama remot) https:(alamat remot)
  • kemudia buka text editor tambahkan file
  • untuk mengecek segala aktifitas di repo bisa ketik di bash , git status
  • akan muncul aktifitas yang telah di buat
  • kemuan segala aktifitas harus di tambahkan dengan mengetik, git add nama file jika hanya satu file
  • tetapi jika banyak, cukup tambahkan git add .(tanda titik)
  • kemudian harus di record atau di commit dengan mengetk
  • git comit -m "menambah file"(keterangahn perubahan yang di buat)
  • kemudian harus di push ke repositori dengan memberi me remote dengan remot yg di buat dengan skrip
  • git push -u origin(nama remot) main/master(branch utama)
  • kemudian file akan di upload ke github
  • pantau kembali aktifitas dengan git status



    • cloning daari repo ke lokal repo

  • buat repo baru
  • kemudian di sebelah kana klik tanda panah pada kode, kemudian copy link nya
  • kemudian buka bash, ketik. git clone https://(link yng di copy) kemudian enter
  • otomatis akan meng import repo dri github ke repo lokal
  • kemudian masuk ke repo yang di import, dengan remot akan otomatis default



    • aktifitas yang sering di lakukan

  • git init - membuat folder jadi repositori git
  • git status - memantau aktifitas
  • git fetch - mengecek bentrokan file yang sama2 di ketik oleh user
  • git push - upload file ke repo github
  • git pull- mengambil file yang bentrok kemudian harus di edit manual
  • git graph - menampilkan aktifitas commit bentuk grafik dengan catatan
  • harus di buat pengenalan, alias graph="git log --all --decorate --oneline --graph"
  • git log - mengecek segala aktifitas commit
  • git commit -am = membuat commit dengan catatan file hasil modify bukan menambah file
  • git remote -v = untuk mengecek link remote
